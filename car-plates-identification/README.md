# Car Plates Identification

This project provides two approaches to extract car license plate numbers from images using Azure AI services. <br>
The input files with car plates are stored in the folder **\pictures**. All the files are generated by [prompt engineering](#prompt-engineering).

## <a name="scripts"></a>Scripts Available

### 1. `plate_reader.py` - Azure AI Foundry (GPT-4o Vision)

Uses Azure AI Foundry's GPT-4o vision capabilities to identify license plates.

- Best for: Intelligent plate recognition with context understanding
- Returns: Plate text only

### 2. `plate_reader_vision.py` - Azure AI Vision (Read OCR)

Uses Azure AI Vision's Image Analysis with Read OCR feature.

- Best for: Precise text extraction with bounding box positions
- Returns: Plate text, position coordinates, confidence scores
- **Generates annotated images** with bounding boxes around detected plates

## <a name="prerequisites"></a>Prerequisites

1. **For `plate_reader.py`**: Azure AI Foundry resource with GPT-4o model deployed
2. **For `plate_reader_vision.py`**: Azure AI Vision (Computer Vision) resource
3. Python 3.8 or later

## <a name="installation"></a>Installation python libraries

Install the required packages in virtual environment:

```powershell
pip install -r requirements.txt
```

## <a name="configuration"></a>Configuration input variables

Edit the `.env` file with your Azure credentials:

```env
# For plate_reader.py (Azure AI Foundry)
AZURE_AI_ENDPOINT=https://your-resource.cognitiveservices.azure.com/openai/deployments/gpt-4o
AZURE_AI_API_KEY=your-api-key
AZURE_AI_MODEL=gpt-4o

# For plate_reader_vision.py (Azure AI Vision)
AZURE_VISION_ENDPOINT=https://your-azure-region.api.cognitive.microsoft.com/
AZURE_VISION_API_KEY=your-api-key
```


## <a name="scripts-usage"></a>Scripts Usage

### Option 1: GPT-4o Vision (plate_reader.py)


### Option 2: Azure AI Vision OCR with Bounding Boxes (plate_reader_vision.py)

## <a name="deployment-scripts"></a>Azure Resource Deployment Scripts

Two PowerShell scripts are provided to automatically deploy the required Azure resources and configure the `.env` file.

### deploy-azure-ai-foundry.ps1

Deploys Azure AI Foundry resources for `plate_reader.py`:

```powershell
.\deploy-azure-ai-foundry.ps1
```

**What it does:**
1. Creates a Resource Group (`rg-plate-reader`)
2. Creates an Azure AI Services account
3. Deploys the GPT-4o model
4. Retrieves endpoint and API key
5. Updates `.env` with:
   - `AZURE_AI_ENDPOINT`
   - `AZURE_AI_API_KEY`
   - `AZURE_AI_MODEL`

**Configuration (at top of script):**
```powershell
$resourceGroupName = "rg-plate-reader"
$location = "eastus"
$deploymentName = "gpt-4o"
```

### deploy-azure-ai-vision.ps1

Deploys Azure AI Vision (Computer Vision) for `plate_reader_vision.py`:

```powershell
.\deploy-azure-ai-vision.ps1
```

**What it does:**
1. Creates a Resource Group (`rg-plate-reader`)
2. Creates an Azure AI Vision (Computer Vision) resource
3. Retrieves endpoint and API key
4. Updates `.env` with:
   - `AZURE_VISION_ENDPOINT`
   - `AZURE_VISION_API_KEY`

**Configuration (at top of script):**
```powershell
$resourceGroupName = "rg-plate-reader"
$location = "eastus"
```

## <a name="considerations-script"></a>Considerations in plate_reader.py

The GPT-4o script includes several optimizations for improved accuracy and reliability:

### 1. Enhanced Prompt Engineering
- Structured system prompt with clear extraction rules
- Requests country/region detection from plate format (e.g., "Italy", "US-California", "EU-Germany")
- Preserves original spacing, hyphens, and formatting
- Zero temperature setting for maximum consistency and reproducibility

### 2. Structured JSON Output
The model returns structured JSON with detailed information:
```json
{
  "plates": [
    {
      "text": "ABC-1234",
      "confidence": 0.95,
      "country": "Italy"
    }
  ],
  "found": true
}
```
- **Confidence scores** (0.0-1.0) for each detected plate
- Automatic fallback to plain text parsing if JSON fails

### 3. Retry Logic with Exponential Backoff
Handles transient API errors automatically:
- **429** (Rate Limit Exceeded) - respects Retry-After header
- **500** (Internal Server Error)
- **503** (Service Unavailable)
- Connection and timeout errors

Retry behavior:
- Maximum 3 retry attempts
- Exponential backoff delays: 1s → 2s → 4s
- Progress messages during retry

### 4. Enhanced Output
- Results display confidence percentage for each detection
- Summary table includes confidence column
- Statistics show average confidence across all successful detections

## <a name="troubleshooting"></a>Troubleshooting with Azure CLI

Use these Azure CLI commands to diagnose and verify your Azure AI Services configuration:

### Check Azure Login Status
```powershell
# Verify you're logged in and see current subscription
az account show
```

### List All AI Services Resources
```powershell
# List all Cognitive Services accounts in your subscription
az cognitiveservices account list --output table

# List accounts in a specific resource group
az cognitiveservices account list --resource-group $rgName --output table

# Get just the account names in a resource group
$cogAccountName =az cognitiveservices account list --resource-group $rgName --query "[].name" --output tsv
```

### Get Resource Details and Endpoint
```powershell
# Get endpoint and custom domain for your resource
az cognitiveservices account show `
    --name $cogAccountName `
    --resource-group $rgName `
    --query "{endpoint: properties.endpoint, customDomain: properties.customSubDomainName}" `
    --output json
```

### List Model Deployments
```powershell
# See all deployed models on your resource
az cognitiveservices account deployment list `
    --name $cogAccountName `
    --resource-group $rgName `
    --output table
```

### Get API Keys
```powershell
# Retrieve the API keys for your resource
az cognitiveservices account keys list `
    --name $cogAccountName `
    --resource-group $rgName `
    --output json

# Get just the primary key
az cognitiveservices account keys list `
    --name $cogAccountName `
    --resource-group $rgName `
    --query "key1" `
    --output tsv
```

### Verify Deployment Details
```powershell
# Get details of a specific model deployment
az cognitiveservices account deployment show `
    --name $cogAccountName `
    --resource-group $rgName `
    --deployment-name "gpt-4o" `
    --output json
```

### Common Issues and Solutions

| Error | Cause | Solution |
|-------|-------|----------|
| **401 Unauthorized** | Invalid API key or wrong endpoint format | Verify API key with `az cognitiveservices account keys list` |
| **404 Not Found** | Wrong endpoint URL format | Use format: `https://<resource>.cognitiveservices.azure.com/openai/deployments/<deployment>` |
| **429 Too Many Requests** | Rate limit exceeded | Wait and retry, or increase SKU capacity |
| **Model not found** | Deployment name mismatch | Check deployment name with `az cognitiveservices account deployment list` |

### Correct Endpoint Format

For the `azure-ai-inference` SDK with Azure Cognitive Services:
```
AZURE_AI_ENDPOINT=https://<your-resource>.cognitiveservices.azure.com/openai/deployments/<deployment-name>
```

Example:
```
AZURE_AI_ENDPOINT=https://ai-plate-reader-1.cognitiveservices.azure.com/openai/deployments/gpt-4o
```
